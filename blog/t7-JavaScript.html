JavaScript vs Ruby

Ruby's Hash vs JavaScript's Object

02/24/2015

I'm brand spanking new to JavaScript but I have some familiarity with Ruby. This week I've gotten my first taste of JavaScript Objects. At first, I was fooled into thinking that these were objects as in 'Object Oriented Programming'. When dealing with an object, being more familiar with Ruby, I thought that what I would be dealing with were classes. Classes in Ruby have attributes, and they are blueprints for objects.

So, when I first read about Objects in JS, properties are mentioned. That lead me to believe that I was defining the equivalent of class attributes in Ruby. This is not the case!

Objects in JS are sort of like Hashes are in Ruby. They are variables that store properties (the Ruby equivalent would be keys), and, when those properties are defined, they correspond with a value.

var object = { property: "value", anotherProperty: "another value", }

Here is the Ruby equivalent:

hash = {k1:"value one", K2:"value two", K3:"value three"}

While in ruby to retrieve the value you would write hash[:k1],

In JS you can write object.property.

You can easily change, add, or delete values to an Object after initializing it.

object.newproperty = "this is a new value"

The equivalent in Ruby looks like this:

hash[k4] = "new value"

If you are familiar with Ruby, you may have noticed that when a property is called on an Object in JS it looks very similar to a method call on an object in Ruby. The truth is that methods can be added to these Objects and called upon through this notation. There are also build in methods in JS that can be called on them. In Ruby, you would need to change the Hash class to add a new instance method. Here is how you add a method to an Object in JavaScript:

object.method = function(num1, num2) { return num1 + num2; }

There is also something called an Object constructor. This functions sort of like a class in Ruby. It is a function that creates Objects in JS. But more on that later!

I hope that this clears some things up about JS. I am an uber-beginner here. But I hope this can shed some light for someone transitioning from Ruby to learning JS for the first time.
