Active Record

03/05/15

Web applications often require a lot of data that is constantly in flux. When dealing with a lot of data, it helps to be able to keep track of that data in an organized way.

Active record is a pattern for facilitating something called Object Relational Mapping, and was originally defined by Martin Fowler in his book Patterns of Enterprise Application Architecture. This means that accessing and storing data about objects in databases is made possible without actually having to go to the trouble of writing SQL queries.

In Ruby on Rails, ActiveRecord is a library that provides a common base for models, and is used to implement the active record pattern.

For example, if you cause a class to inherit from the class Base in the ActiveRecord library, you gain access to methods that allow you to create and pull data from a database without writing any SQL. Methods in ActiveRecordâ€™s classes allow one to easily turn class attributes into columns in SQL and new objects into rows.

class User < ActiveRecord::Base

To create the table in the database, it is necessary to create a schema class that inherits the tools from a class in ActiveRecord called Migration. This may be confusing, remember that once you create a class that inherits from Base, you gain methods to read and write new rows and columns in a data base, and that is not the same as actually making a new table.

class UserTable < ActiveRecord::Migration

Again, this is all done through writing easy-to-read Ruby, making the life of the programmer a lot easier.
